Analyze Requirements

Đưa cả cục requirements — nhận về backlog sprint-ready, tạo thẳng lên tracker.

❌ Vấn đề:Nhận doc requirements dài dằng dặc rồi ngồi chẻ tay thành từng ticket — title, acceptance criteria, story point, dependency… làm đi làm lại, hết cả buổi.

✅ AIDLC:Đóng vai business analyst, chẻ doc thành task sprint-sized đầy đủ trường, rồi tạo thẳng lên Jira / GitHub / Linear / Redmine.

CŨ Chẻ ticket bằng tay
1. Đọc hết doc requirements dài
2. Tự viết từng ticket một
3. Thêm AC, story point, priority, label — lặp đi lặp lại
4. Sót dependency giữa các task → dính giữa sprint
5. Copy-paste lên Jira, rồi doc sau lại làm lại từ đầu
vs
MỚI Analyze Requirements
1. Trỏ vào 1 file / URL / text — hoặc kéo từ Jira, GitHub, Linear, Redmine
2. Đóng vai senior business analyst, chẻ ra task sprint-sized
3. Mỗi task đủ: mô tả, AC (Given/When/Then), priority, story point, label, dependency
4. Đối chiếu issue cũ → không tạo trùng
5. ✓ Tạo thẳng lên tracker — hoặc export ra local
