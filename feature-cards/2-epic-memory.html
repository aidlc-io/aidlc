Epic Memory auto-load

Mỗi epic có một "bộ nhớ" gọn — mở lại là agent nhớ ngay, không đốt token đọc lại từ đầu.

❌ Vấn đề:Quay lại một epic sau vài ngày (hoặc đổi agent), phải đọc lại toàn bộ artifact + git history mới hiểu context → tốn cả đống token, dễ quên quyết định cũ.

✅ AIDLC:Một digest nhỏ epic-memory.json tự được nạp vào context mỗi khi prompt nhắc tới epic đó.

CŨ Mỗi lần đều đọc lại
1. Mở lại epic cũ — quên sạch context
2. Đọc lại từng artifact.md + lịch sử git
3. Đốt token khủng chỉ để "nhớ lại"
4. Đổi agent = mất hết, làm lại từ 0
5. Lặp lại sai lầm cũ vì không nhớ quyết định trước
vs
MỚI Nhớ tự động
1. Bật toggle Memory auto-load: On ở đầu list Epics
2. Hook UserPromptSubmit theo dõi prompt của bạn
3. Prompt nhắc tới epic → tự chèn epic-memory.json vào context
4. Vừa làm vừa ghi thêm decision / constraint quan trọng
5. Annotation rounds cũng tự append context vào memory
6. ✓ Agent nào mở cũng hiểu ngay — cực rẻ token

Opt-in: không bật thì không có gì chạy — toggle chỉ thêm/bớt 1 hook trong settings.json
