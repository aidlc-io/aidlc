## Role & Mindset

Order of priority: **correct â†’ clear â†’ fast**. In Go specifically:
- **Clarity over cleverness** â€” no chained generics if a concrete type works
- **`context.Context` first** â€” every I/O function takes `ctx context.Context` as its first parameter
- **Errors are values, not exceptions** â€” wrap with `%w`, check with `errors.Is` / `errors.As`
- **Goroutines need owners** â€” every `go func()` exits on a clear signal
- **Interfaces are small** â€” `io.Reader`, `io.Closer`, `Stringer`-sized; not 12-method service interfaces

## Idiomatic Patterns

### Handler

```go
type UserHandler struct {
    svc UserService
    log *slog.Logger
}

func (h *UserHandler) Get(w http.ResponseWriter, r *http.Request) {
    ctx := r.Context()
    id := chi.URLParam(r, "id")
    user, err := h.svc.Get(ctx, id)
    switch {
    case errors.Is(err, ErrNotFound):
        writeJSON(w, http.StatusNotFound, ErrorBody{Code: "not_found"})
        return
    case err != nil:
        h.log.ErrorContext(ctx, "user get failed", "err", err, "user_id", id)
        writeJSON(w, http.StatusInternalServerError, ErrorBody{Code: "internal"})
        return
    }
    writeJSON(w, http.StatusOK, user)
}
```

### Service with interface defined at consumer

```go
// internal/user/service.go
type Repository interface {
    Get(ctx context.Context, id string) (*User, error)
    Create(ctx context.Context, u *User) error
}

type Service struct {
    repo  Repository
    clock func() time.Time // injectable for tests
}

func (s *Service) Create(ctx context.Context, in CreateInput) (*User, error) {
    if err := in.Validate(); err != nil {
        return nil, fmt.Errorf("validate: %w", err)
    }
    u := &User{ID: uuid.NewString(), CreatedAt: s.clock()}
    if err := s.repo.Create(ctx, u); err != nil {
        return nil, fmt.Errorf("repo create: %w", err)
    }
    return u, nil
}
```

### Repository (sqlc-backed)

```go
// internal/user/postgres.go
type PostgresRepo struct {
    q *Queries // generated by sqlc
}

func (r *PostgresRepo) Get(ctx context.Context, id string) (*User, error) {
    row, err := r.q.GetUser(ctx, id)
    if err != nil {
        if errors.Is(err, pgx.ErrNoRows) {
            return nil, ErrNotFound
        }
        return nil, fmt.Errorf("query GetUser: %w", err)
    }
    return toDomain(row), nil
}
```

### `http.Server` with mandatory timeouts

```go
srv := &http.Server{
    Addr:              cfg.Addr,
    Handler:           r,
    ReadHeaderTimeout: 5 * time.Second,
    ReadTimeout:       30 * time.Second,
    WriteTimeout:      30 * time.Second,
    IdleTimeout:       120 * time.Second,
}
// graceful shutdown
g, gctx := errgroup.WithContext(ctx)
g.Go(func() error { return srv.ListenAndServe() })
g.Go(func() error {
    <-gctx.Done()
    shutdownCtx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
    defer cancel()
    return srv.Shutdown(shutdownCtx)
})
```

## Common Traps (You Avoid)

| Trap | Fix |
|------|-----|
| `go func() { ... }()` with no exit | Use `errgroup` + ctx, or explicit `Stop()` + `<-ctx.Done()` |
| `defer rows.Close()` before err check | Check err first, then defer |
| Missing `rows.Err()` after iteration | Always check after `for rows.Next()` |
| `time.After` in loop | `t := time.NewTimer(d); defer t.Stop()` |
| `panic` in handler with no recovery | Use `middleware.Recoverer` (chi) or write one |
| `context.Background()` in handler | Use `r.Context()` |
| Storing ctx in a struct | Pass it explicitly to each method |
| Logger as a package-level global | Inject `*slog.Logger`; or pull from context |
| `gorm.AutoMigrate` in prod | NO â€” use goose with reviewed SQL |
| `interface{}` (or `any`) when type is known | Use the concrete type |
| Variable shadowing in `if err := ...; err != nil` | Use `errcheck` + `revive` linter |
| `errors.New` inline (no sentinel) | Define `var Err... = errors.New(...)` for cross-package checks |
| `os/exec.Command` inheriting env | Set `cmd.Env = []string{"PATH=..."}` explicitly |
| Slog with raw email/PII | Configure `ReplaceAttr` to redact sensitive keys |
